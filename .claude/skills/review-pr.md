現在の変更（または指定されたブランチ・PR）を 3 つの独立したレビュワーエージェントで並行レビューし、結果を統合してレポートを作成してください。

## 手順

### 1. レビュー対象の確認

```bash
git diff main...HEAD --stat        # 変更ファイル一覧
git log main...HEAD --oneline      # コミット一覧
```

変更の概要を把握する。

### 2. 3 エージェントを並行起動

以下の 3 エージェントを **同時に（1 つの Agent tool 呼び出しではなく、単一メッセージで複数の Agent tool を呼び出して）** 起動する：

1. **code-reviewer** エージェント
   - コードの正確性・TypeScript 型安全性・既存パターンとの一貫性をレビュー

2. **security-reviewer** エージェント
   - 認証・認可・XSS・インジェクション・Cookie 設定をレビュー

3. **test-reviewer** エージェント
   - テストカバレッジ・テスト品質・エッジケースの網羅性をレビュー

各エージェントには「変更ファイルの差分を確認し、担当観点でレビューして問題点と推奨改善を報告せよ」と指示する。

### 3. レビュー結果の統合

3 エージェントの結果が揃ったら、以下の形式でレポートを作成する：

---

## PR レビューレポート

### 変更概要
（変更ファイル数・変更行数・目的の要約）

### コードレビュー結果 🔍
（code-reviewer の結果）

### セキュリティレビュー結果 🔒
（security-reviewer の結果）

### テストレビュー結果 ✅
（test-reviewer の結果）

### 総合判定
- **LGTM** / **要修正** / **要議論**

### 必須対応事項
（マージ前に必ず対応すべき項目）

### 推奨改善事項
（対応推奨だが必須ではない項目）

---

問題があれば具体的な修正案を提示し、ユーザーに確認を求めてから修正を進める。
